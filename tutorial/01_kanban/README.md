# チュートリアル1: Kanbanボード

## 概要

このチュートリアルでは、シンプルなKanbanボードを題材に
**AIを使った開発フロー全体**を一通り体験する。

BDD・DDDの深い理解よりも「要求定義 → 詳細設計 → 実装 という流れで
どう開発が進むか」を体感することが目的。

---

## 学習目標

このチュートリアルを終えると以下ができるようになる。

- [ ] AIとの壁打ちで要件を言語化する流れを体験できる
- [ ] 粗い振る舞いシナリオ（Given/When/Then）を書き出せる
- [ ] 要求定義に必要なファイルを作れる
- [ ] Gherkin形式のfeatureファイルを基本的に書ける
- [ ] ユビキタス言語を定義できる
- [ ] ADRで技術的判断を記録できる
- [ ] 要求定義を渡して詳細設計を生成させられる
- [ ] 詳細設計を渡して実装させられる
- [ ] Playwrightの基本的なE2Eシナリオを書ける

---

## 題材: Kanbanボード

### 作るもの

カード（タスク）をカラム（状態）間で移動できるシンプルなKanbanボード。

```
[ Todo ] → [ In Progress ] → [ Done ]
  カードA      カードB          カードC
  カードD
```

### なぜKanbanボードか

- 誰でも使い方を知っている（ドメイン知識の壁がない）
- フロントエンド（ドラッグ操作）とバックエンド（状態管理）の両方が出てくる
- BDD/DDDの概念が綺麗にマッピングできる

---

## このチュートリアルで導入するプラクティス

| プラクティス | 説明 |
|------------|------|
| 壁打ちによる要件言語化 | AIと対話しながら要件と粗いシナリオを言語化する |
| 実例マッピング（Example Mapping） | 壁打ちメモからルール・例・疑問を構造化する発見手法 |
| プロダクト要件概要（overview.md） | What + Why + スコープ外を記述する |
| featureファイル | 振る舞いをGherkin形式で記述する（バックエンド用・フロントエンド用） |
| ユビキタス言語 | プロジェクト内の言葉を統一する |
| ADR | 技術的判断を記録する |
| 詳細設計の生成 | 要求定義をAIに渡して詳細設計を作る |
| 実装 | 詳細設計をAIに渡して実装する |
| Playwright（基礎） | E2Eシナリオの正常系を書く |

---

## チュートリアルの章構成

```
第1章: 壁打ちで要件を言語化する
  └─ Kanbanボードの「何を作るか・なぜ作るか」をAIとの対話で固める
  └─ 実例マッピングでルール・例・疑問を整理する
  └─ 実例マッピングの成果をGiven/When/Thenシナリオに変換する
  ゴール: 要件と粗いシナリオが言語化されている状態

第2章: 要求定義を整える
  └─ 第1章の成果を 要件概要 · 振る舞い定義 · ドメイン定義 · 判断記録 のフォーマットに落とす
  └─ ユビキタス言語を定義する
  └─ 技術選定をADRに記録する
  ゴール: 要求定義が一通り揃っている状態

第3章: 詳細設計を生成する
  └─ 要求定義をAIに渡して詳細設計を生成させる
  └─ 生成結果をレビュー・修正する
  ゴール: AIが実装に着手できる詳細設計がある状態

第4章: 実装する
  └─ 詳細設計をAIに渡してバックエンドを実装する
  └─ フロントエンドを実装する
  └─ featureファイルのシナリオを受け入れ条件として使う
  ゴール: シナリオを満たす動くアプリケーションがある状態

第5章: Playwrightでテストする
  └─ フロントエンド用のGherkinシナリオを書く
  └─ Playwrightで正常系E2Eテストを実装・実行する
  ゴール: 正常系E2Eテストが通る状態

振り返り
  └─ 何を学んだか
  └─ チュートリアル2で何が加わるか
```
