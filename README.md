# AI × BDD/DDD 開発標準

AIを用いた開発の標準プロセスとチュートリアル。

「人間が判断し、AIが実行する」協調開発の思想・プロセス・実践をまとめたリポジトリ。

---

## 構成

```
guide/       ← 思想・プロセス・リファレンス（なぜ・どう進めるか）
tutorial/    ← ハンズオンチュートリアル（手を動かして学ぶ）
```

### tutorial/

3つのサンプルプロジェクトを通じて段階的に学ぶハンズオンシリーズ。

| チュートリアル | 題材 | 主な学習内容 | 難易度 |
|--------------|------|------------|--------|
| `01_kanban/` | Kanbanボード | 開発フロー全体の体験 | ★☆☆ |
| `02_restaurant/` | レストラン注文システム | イベントストーミング・ポリシー・異常系 | ★★☆ |
| `03_reservation/` | 予約システム | 集約・競合制御・コンテキスト分割 | ★★★ |

---

## 始め方

1. `guide/01_philosophy.md` を読んで全体思想を理解する
2. `tutorial/01_kanban/` を最初から最後まで通しでやる
3. `guide/appendix/` を参照しながら `tutorial/02_restaurant/` へ
4. `tutorial/03_reservation/` で全プラクティスを統合する

---

## 核心となる考え方

```
ドメイン知識を持つ
  └─ 言語化して壁打ちする
       └─ 設計ドキュメントに落とす
            └─ 適切な粒度でAIに委譲する
                 └─ レビューして次の判断をする
```

このサイクルを回し続けることが、AIとの協調開発の本質である。
